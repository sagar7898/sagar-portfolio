Invoice AI Automation – Case Study
Problem

Businesses receive hundreds of invoices through email every day. Manually opening emails, downloading attachments, and extracting invoice data is time-consuming and prone to human errors.

System Architecture
Email Inbox
↓
ASP.NET Backend
↓
Azure AI
↓
Data Extraction
↓
Database Storage
Approach

Developed an automated pipeline using ASP.NET and Microsoft Graph API to read incoming emails and identify invoice attachments. The documents are processed using Azure AI Document Intelligence to extract structured invoice data.

Solution

Implemented an intelligent document processing system that automatically reads invoice attachments from email, sends them to Azure AI for analysis, and stores the extracted data in structured format for reporting and automation workflows.

  • Microsoft Graph API for Email Integration
  • Azure AI Document Intelligence
  • ASP.NET Backend Processing
  • SQL Database Storage
Result

The automation reduced manual data entry significantly. Invoice information can now be processed automatically and made available instantly for reporting and accounting workflows.

Technical Challenge

Handling different invoice formats was challenging. Azure AI Document Intelligence models were used to accurately extract fields from multiple layouts while maintaining reliable backend processing.

WhatsApp Automation – Case Study
Problem

Businesses needed a way to send invoices and documents directly through WhatsApp from the application dashboard. Sending files manually through WhatsApp Web was slow and inefficient.

System Architecture
User Action
↓
ASP.NET Backend
↓
PuppeteerSharp
↓
WhatsApp Web
↓
File Sent
Approach

Designed an automation workflow using PuppeteerSharp to control WhatsApp Web through an ASP.NET backend service.

Solution

Implemented PuppeteerSharp automation that opens WhatsApp Web, handles QR authentication, searches contacts, uploads files, and sends messages automatically.

  • ASP.NET Backend
  • PuppeteerSharp Automation
  • WhatsApp Web Integration
Result

Users can now send invoices and documents directly from the application dashboard, reducing manual effort and improving workflow efficiency.

Technical Challenge

WhatsApp Web uses dynamic DOM elements. Stable selectors and wait logic were implemented to reliably detect the send button and file upload input during automation.

Task Manager System – Case Study
Problem

Teams often struggle to track customer-related tasks efficiently. Tasks were being managed manually or across multiple tools, making it difficult to monitor progress, deadlines, and task ownership.

System Architecture
User Dashboard
↓
MVC Controller
↓
Service Layer
↓
Repository Layer
↓
SQL Database
Approach

Built a task and customer management system using ASP.NET Core MVC with a layered architecture to maintain separation of concerns between controllers, services, and repositories.

Solution

Developed a responsive dashboard that allows users to create, manage, and monitor tasks linked to customers. The system includes validation, filtering, task status updates, and automated reminders.

  • Customer Management Module
  • Task Creation and Tracking
  • Email Reminder Automation
  • Dashboard with Task Statistics
  • Excel Export Functionality
Result

The application provides a centralized system to manage tasks and customer activities efficiently, improving task visibility and ensuring important deadlines are not missed.

Technical Challenge

Implementing automated email reminders and maintaining a clean layered architecture required careful separation of business logic, repository queries, and controller actions.
